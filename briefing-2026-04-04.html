Sofort Blocker

Muss vor allen anderen Features passieren — alles Weitere baut darauf auf.

  • MedicalEvent Datenmodell vereinheitlichen Kritisch
    Impfungen und Behandlungen teilen sich aktuell kein gemeinsames Datenmodell. Bevor die Gesundheits-UX sinnvoll weitergebaut werden kann, braucht es ein einheitliches MedicalEvent Interface. Betrifft DB-Schema, TypeScript-Typen und Supabase-Queries.
    Umfang: Gross Team: DB Expert + Developer Risiko: Schema-Migration
  • DataContext + Screens auf MedicalEvent umbauen Kritisch
    Nach dem Datenmodell-Umbau muessen DataContext (~350 Zeilen God-Object) und alle Gesundheits-Screens das neue Modell nutzen. Idealerweise gleich das M7-Refactoring (Context aufsplitten) mitmachen.
    Umfang: Gross Team: Developer Abhaengig von: MedicalEvent Datenmodell

Vor Release Release-Blocker

Muss gefixt sein bevor die App an echte Nutzer geht. Sortiert nach Risiko.

  • F-02: togglePro() absichern — IAP implementieren Security
    Premium-Status kann aktuell clientseitig getoggelt werden. Braucht echte In-App-Purchase Anbindung und RLS-Policy fuer is_premium.
    Umfang: Gross Team: Developer Risiko: Umsatzverlust ohne IAP
  • F-03: CORS in Edge Function einschraenken Security
    Edge Function erlaubt aktuell Origin: *. Muss auf die App-Domain eingeschraenkt werden.
    Umfang: Klein Team: Developer
  • F-07: DB-Fehler in DataContext.refresh() nicht verschlucken Bug
    Wenn Supabase-Queries in refresh() fehlschlagen, sieht der User einfach keine Daten — ohne Fehlermeldung. Silent Failures.
    Umfang: Klein Team: Developer
  • F-05/F-06: goBack() bei null-Events/Pets absichern Bug
    EventDetailScreen und PetDetailScreen crashen wenn sie mit einer ungültigen ID navigiert werden (z.B. geloeschtes Pet/Event). Braucht Null-Checks + Fallback-Navigation.
    Umfang: Klein Team: Developer
  • F-08: Pet-Fotos in Supabase Storage Feature
    Pet-Fotos werden aktuell nicht persistiert. Gleiche Logik wie Dokumente nutzen: Upload in Storage, Pfad in DB, signed URLs on-demand.
    Umfang: Mittel Team: Developer
  • F-11: notification_id ins Schema nachpflegen Schema-Drift
    Die Spalte notification_id existiert in der Live-DB, ist aber nicht in supabase-schema.sql dokumentiert. Schema-Drift-Risiko bei Re-Setup.
    Umfang: Klein Team: Developer / DB Expert
